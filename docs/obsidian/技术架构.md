# 技术架构

## 技术栈

| 层 | 技术 |
|---|------|
| 后端 | Node.js + Express 4.21 |
| 数据库 | SQLite (better-sqlite3)，WAL 模式 |
| 前端 | 原生 HTML/CSS/JS（无框架） |
| 认证 | bcryptjs + JWT |
| 安全 | helmet (CSP) + express-rate-limit |
| 部署 | Render.com 免费层 |

## 部署地址

- **生产环境：** https://suriorder.onrender.com
- **注意：** 免费层 15 分钟无请求后休眠，冷启动 30-50 秒

## 项目结构

```
SuriOrder/
├── server.js          # 主入口
├── db/schema.js       # 建表 + 迁移 + demo 种子
├── middleware/         # auth.js, rateLimit.js
├── lib/               # sanitize, events, backup, logger, whatsapp
├── routes/            # auth, shops, orders, admin, webhook
├── public/            # index, order, admin, privacy, tos, shared.{css,js}
└── tests/             # 40 个 API 测试
```

## 环境变量

| 变量 | 说明 |
|------|------|
| `JWT_SECRET` | 未设置时自动生成 |
| `WHATSAPP_TOKEN` | Meta Cloud API token（可选） |
| `WHATSAPP_PHONE_ID` | 发消息用的 ID（可选） |
| `NODE_ENV` | production 时启 HTTPS 重定向 |
| `ALLOW_QUERY_TOKEN` | SSE 用，设为 '1' |

## 本地运行

```bash
npm run dev    # 开发（热重载）
npm test       # 40 个测试
npm start      # 生产
```

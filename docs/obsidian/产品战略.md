# 产品战略

## 核心认知

**SaaS 不是产品，SaaS 是数据管道。** 真正的产品是建在餐厅网络之上的 B 端订单引擎。

> "你这个倒有意思，我本来觉得这算是两个项目，你这样就是合在一起来做了"
> — Arthur，意识到 SaaS 和 B 端团餐是一个链条

---

## 三层架构

### 第一层：SaaS（永远免费）

| 维度  | 决策                   |
| --- | -------------------- |
| 定价  | 免费，永久                |
| 目的  | 降低餐厅决策门槛，快速铺网络       |
| 换什么 | 菜单数据、订单数据、餐厅关系       |
| 类比  | WhatsApp 不收钱，因为是通讯入口 |

**不收钱的原因不是收不到，是收费会减慢网络增长速度，而网络规模决定后面能赚多少。**

### 第二层：B 端团餐（第一收入）

**模式：** 企业找你要团餐 → 你匹配给网络里的餐厅 → 抽 10-15%

**为什么你能做，单个餐厅做不了：**

| 企业要的 | 单个餐厅 | 你的网络 |
|---------|---------|---------|
| 每天不重样 | 做不到 | 10 家轮换 |
| 多种菜系 | 只有一种 | 中餐+印尼+印度菜 |
| 统一月结对账 | 老板懒得搞 | 你出一张发票 |
| 出了问题有人负责 | 推给你 | 你承担 |

**算账（10 家店、2 个企业客户）：**
- 日均 60 份 × SRD 80 × 15% = SRD 720/天
- 月收入 ≈ SRD 15,840（超过苏里南人均月收入）

### 第三层：品牌自营（终局）

数据告诉你哪里需求最密、什么菜最火、什么价格最优 → 放自己的餐车/取餐点。

- 位置靠数据选，不是猜
- 菜单靠数据定，不是凭感觉
- 毛利 100%，不跟餐厅分

## 为什么不跟外卖公司正面竞争

| | 外卖公司 | SuriOrder |
|---|---------|-----------|
| 配送 | 自建骑手，重资产 | 无配送，到店取 |
| 抽成 | 20-30% | 0%（SaaS 层） |
| 立场 | 抢餐厅流量 | 帮餐厅做增量 |
| 关系 | 博弈 | 合作 |

**核心差异：** "你是帮他们的自己人，不是抽成的平台。"

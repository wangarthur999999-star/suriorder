# 产品功能路线图

## P0 — 演示前必须有

| 功能                 | 状态    | 说明       |
| ------------------ | ----- | -------- |
| 顾客下单 → WhatsApp 通知 | ✅ 已上线 | 核心链路     |
| 后台订单管理             | ✅ 已上线 | 状态流转     |
| 菜单 + 分类管理          | ✅ 已上线 | CRUD     |

| 四语言 (nl/en/zh/es)  | ✅ 已上线 | 5/19 完成  |

## P0 — 第一家店试用期间加

| 功能 | 复杂度 | 为什么 |
|------|--------|--------|
| 已售罄一键标记 | 低 | 老板每天用，感知最强 |
| 熟客下单次数统计 | 低 | 数据已有，展示出来就行 |

## P1 — 有价值的差异化

| 功能 | 复杂度 | 为什么 |
|------|--------|--------|
| "再点一次"按钮 | 中 | 顾客端留存钩子，上班族杀手锏 |
| 月结对账单 | 中 | 苏里南熟人赊账文化，华人老板实用 |
| 智能菜单排序 | 低 | 按销量排，新客看到"大家都爱点的" |

## P2 — 数据驱动型

| 功能 | 前提 |
|------|------|
| 自动建议"今日特价"清库存 | 有销售数据积累 |
| 节日菜单推荐 | 有季节数据 |
| 新店选址定价咨询 | 有区域数据 |

## P3 — 有量了再说

| 功能 | 前提 |
|------|------|
| LLM 翻译补齐 | 需要 API key + 足够需求 |
| 供应链团购 | 50+ 家餐厅 |

## ❌ 砍掉的

| 功能 | 原因 |
|------|------|
| 转单协作 | 餐厅是竞争关系，不会转单给别人 |
| Sranan Tongo | 本地西班牙语需求更强 |

## 功能优先级原则

> **他提的需求 > 你计划的功能。**

第一家店用两周后，他说的痛点才是真的痛点。闭门造的功能大概率用不上。
